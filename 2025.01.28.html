Block and Inline elements

Block Elements

A block element starts on a new line and stretches to the full width avvailable, pushing any adjacent content below them.
Block elements stack vertically
Some examples of block elements are:

  • < div >
  • < p >
  • < h1 > to < h6 >
  • < ul >, < ol >, < li >
  • < table >, < tr >, < td >
  • < form >
  • < header >, < footer>, < section>, < article>

Inline Elements

Inline elements are HTML elements that do not create a new block in the document flow.
Inline elements flow tihin the content and only take up as much width as necessary.
Some examples of inline elements are:

  • < span >
  • < a >
  • < strong >,< em >
  • < img >
  • < i >, < b >

Semantic HTML

Semantic: relating to meaning in language or logic

Non-semantic tags

  • Tells nothing about it's content
  • Solely for styling pruposes
  • The name doesn't suggest meaning, or value

<span>: an inline-level element commonly used to identify smaller groupings of text within a block-level element.

We'll commonly see these tags used with class or id attributes for styling purposes.

Attributes

The Class Attribute "Class-name" specifies one or more class names for an element.

  • The class attribute is mostly used to point to a class in a style sheet, and JavaScript (via the HTML DOM).
  • You can use class multiple times on a page.
The ID Attribute "id-name" specifies a unique id for an HTML element.
  • The value must be unique within the HTML document



Semantic HTML

  • HTML elements that accurately describe the purpose of the content on a web page, rather than just its presentation.
  • Improves website accessibility to screen readers or any assistive device that scans the web page for elements with meaning.
  • Helps with collaboration and site maintenance.
